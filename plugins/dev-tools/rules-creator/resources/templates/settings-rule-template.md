# ルール名

## 適用範囲（Paths）

このルールは以下のファイルパターンに適用されます：

```yaml
paths: src/**/*.ts, tests/**/*.test.ts
```

**Pathsパターン構文：**

- `**` - 任意の深さのディレクトリ（例：`src/**/*.ts` は src 配下の全 TypeScript ファイル）
- `*` - 任意のファイル名（例：`*.md` はプロジェクトルートの全マークダウンファイル）
- `{a,b}` - ブレース展開で複数パターン（例：`{src,lib}/**/*.ts`）
- カンマ区切り - 複数パターンを組み合わせ（例：`src/**/*.ts, tests/**/*.test.ts`）

**Note:** `paths` フィールドを省略すると、ルールは全ファイルに無条件適用されます。特定のファイルタイプやディレクトリに限定したい場合のみ指定してください。
